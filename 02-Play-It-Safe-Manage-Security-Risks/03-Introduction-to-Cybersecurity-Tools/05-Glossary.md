# Introduction to Cybersecurity Tools
# Glossary 

| **Term** | **Definition** |
|--------|---------------|
| **Chronicle** | A cloud-native SIEM tool designed to retain, analyze, and search large volumes of security data. |
| **Incident Response** | An organization’s rapid process to identify a security attack, contain the damage, and recover from the effects of a security breach. |
| **Log** | A record of events that occur within an organization’s systems, networks, or applications. |
| **Metrics** | Key technical measurements—such as response time, availability, and failure rate—used to assess the performance of a software application or system. |
| **Operating System (OS)** | The interface between computer hardware and the user that manages system resources and runs software applications. |
| **Playbook** | A documented guide that outlines step-by-step actions for handling specific operational or security situations. |
| **Security Information and Event Management (SIEM)** | An application that collects and analyzes log data to monitor critical activities and detect security threats in an organization. |
| **Security Orchestration, Automation, and Response (SOAR)** | A collection of tools, applications, and workflows that use automation to respond to security events with minimal human intervention. |
| **SIEM Tools** | A software platform that collects, analyzes, and correlates security data from multiple sources to detect threats, investigate incidents, and support compliance. |
| **Splunk Cloud** | A cloud-hosted SIEM tool used to collect, search, and monitor log data. |
| **Splunk Enterprise** | A self-hosted SIEM tool used to retain, analyze, and search log data while providing real-time security alerts. |
