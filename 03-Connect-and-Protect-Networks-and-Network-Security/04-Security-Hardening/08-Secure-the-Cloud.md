## üéØ What Is Cloud Computing?

### üìå Definition

**Cloud computing** is a model that provides:

* Convenient, on-demand network access
* A shared pool of configurable computing resources

These resources can be:

* Quickly provisioned
* Released with minimal management effort
* Accessed with little interaction from the service provider

---

## üîê Why Cloud Infrastructure Must Be Secured

Just like traditional IT infrastructure, **cloud infrastructure must be secured**.

Organizations often worry about:

* Data privacy
* Resource protection
* Unauthorized access

These concerns are addressed through:

* Cryptography
* Secure configurations
* Access controls

---

## ‚öôÔ∏è Cloud Security Considerations

Cloud computing provides:

* Speed of deployment
* Cost savings
* Scalability

But it also introduces **unique security challenges**.

---

## üë§ Identity Access Management (IAM)

### üìå Definition

**IAM** is a set of processes and technologies used to:

* Manage digital identities
* Control user access to cloud resources

---

### ‚ö†Ô∏è Common IAM Risk

* Loosely configured user roles
* Excessive permissions

üîê **Impact:**

* Unauthorized users may gain access to critical cloud operations

> Proper IAM configuration is critical to cloud security.

---

## ‚öôÔ∏è Configuration Management

The cloud ecosystem is **highly complex**.

Security challenges include:

* Precise configuration of each service
* Secure cloud migrations
* Continuous configuration management

---

### ‚ö†Ô∏è Misconfiguration Risk

* Misconfigured cloud services are a **leading cause of breaches**
* Errors during migration or updates can expose vulnerabilities

üß† Careful attention by administrators and architects is essential.

---

## üéØ Attack Surface in the Cloud

### üìå Definition

An **attack surface** is the total number of potential entry points attackers can exploit.

---

### Cloud-Specific Considerations

* CSPs offer many low-cost services
* Each service adds potential risk
* More services = larger attack surface

---

### üîê Important Insight

If designed correctly:

* Multiple cloud services **do not automatically increase entry points**

However:

* Poor design can expose many vulnerabilities

---

## ‚ö†Ô∏è Zero-Day Attacks

### üìå Definition

A **zero-day attack** exploits a vulnerability that is previously unknown.

---

### Cloud Advantage

* CSPs often detect zero-day attacks earlier
* CSPs can:

  * Patch hypervisors
  * Migrate workloads to other VMs

This helps protect customers from impact.

---

### Organization Responsibility

* Apply OS-level patches
* Use available patching tools

---

## ‚öñÔ∏è Shared Responsibility Model

### üìå Definition

The **shared responsibility model** defines security responsibilities between:

* Cloud Service Provider (CSP)
* Organization using the cloud

---

### üè¢ CSP Responsibilities

* Physical data centers
* Hypervisors
* Host operating systems

---

### üßë‚Äçüíº Organization Responsibilities

* Data stored in the cloud
* Applications
* User access and IAM
* Configuration of services

---

### ‚ö†Ô∏è Common Mistake

Assuming the CSP handles **all** security.

> CSP secures the **cloud**, not what you put **in** the cloud.

---

## üß† Key Takeaways 

* Cloud introduces unique security challenges
* IAM misconfigurations are high risk
* Misconfiguration is a leading cause of breaches
* CSPs help with zero-day mitigation
* Security is a shared responsibility
